HTML Basics

HTML stands for Hyper-Text Markup language. Marked up language gives more value to regular text. HTML is the core tech on the web. Every single page is an HTML page. Browsers understand how to read the tags. HTML is the universal standard for how web pages work. As a markup language, there will be a portion of the page that isn't rendered.

<Tags>


"You're it. "

Names surrounded by opening and closing tags are rendered. Most but not all having opening and closing tags. Tags may also contain attributes. These attributes usually contain "" marks and rest between the open and close brackets. In HTML, quoting works with both single or double quotes. However, they MUST be paired. i.e. "ABC" or 'ABC' both work.

<!doctype html>

This is preamble. Starts at the beginning of each page.

<html lang ="en">

The root of an HTML document. All other tags must exist within the HTML tags. May contain only one head and one body in that order.

<head>

Contains metadata about the document. Must contain the TITLE tag. May include information about:

  • author
  • description
  • css
  • javascript

<body>

Contains the visible elements of the html document. There can only be ONE BODY tag.

<Tags cont...>

Tags are BLOCK or INLINE

Inline Elements


"Close but no cigar."

Inline elements are rendered to flow right along with any other elements. No space is added above, below, right or left. Examples include:

  • <a>: anchor tag, AKA "link". Renders a string that, when clicked, transfers to another url. Requires href atttribute. Content is the text to display on the page.
  • <button>: button tag renders a button containing text
  • <img>: renders an image, image is specified with SRC attribute. Size can be modified with heigh and width
  • <input>:a form tag. Used mostly to render an input box. Can render a number of different form elements based on the type attribute
  • <select, option>: select from options. Renders a list or dropdown list with a defined set of options. Options are defined using OPTION tags within the SELECT tags. Options can use the value attribute.
  • < span>: group collection of data and elements. Provides the ability to apply a sstyles to a group of data and/or elements. Used to denote emphasis and change how text is rendered.

<block>

Different than an inline element.

<div >

Div - group elements, similar to a span tag, but div is a block element

<form>

Form is a collection of user input tags. Attribute method = get or post, attribute action = url

<ol, ul, li>

Ordered and unordered lists, create lists with either bullets, or numbers. Items are declared with the <li> tag.

<p>

Paragraph, renders content inline with a blank line above and below

<table>

Renders text in rows and columns. THEAD defines the table headers. TBODY defines the table data. TR defines the rows. TD defines one column.

LabelData

<Attributes>

Attributes add more info or IDs to an html tag. Values for attributes should be enclosed in quotes. Common attributes:

  • id
  • style
  • class
  • value

id attribute

provides a unique identifier for any tag it is applied to. The identifier must be unique among all other indentifiers on this html document. Very useful to programmatically access a particular tag

style attributes

provides the ability to attach CSS styles to a tag

class attribute

provides the ability to assign a CSS class style to a tag. The class style must be defined in an included stylesheet. Multiple class styles are separated by a space and are applied in order of appearance.

value attribute

Provides the ability to get and set the contents of a form tag

Events

Events are the way to respond to user interactions with the html page. To respond to an event, the event must be added to the tag along with the code to execute when the event occurs. Common events:

  • OnClick
  • OnLoad

Fonts

  • Size: 8 pt, 10 pt, 12 pt...(16 pt = 1.5 em)
  • Families [Arial, Times New Roman, Courier New]
  • Weight[Bold]
  • Caps
  • Kerning
  • Italics

Borders

Bounding box outside the text content. Multiple ways to specify the box.

border: 1px// all four sides

border: 1px solid black;

Margin & Padding

Spacing outside the borders, usually expressed in "px" or "em"
